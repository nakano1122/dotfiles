## 基本方針

- このファイルはすべてのプロジェクトで共通に適用される作業ルールを定義する。
- プロジェクト固有の仕様・技術・制約は、各プロジェクトの `CLAUDE.md` を優先する。


## コミュニケーション

- すべての対話は日本語で行う。
- 要求・仕様・前提が曖昧な場合、悩みすぎず早い段階でユーザーに確認する。
- 推測や仮定で進める場合は、その旨を明示する。


## 作業の進め方

- 作業は可能な限り小さな単位に分割する。
- 以下のタイミングで簡潔に進捗を報告する：
  - 作業開始時
  - 中間成果が出た時
  - 作業完了時
- 完璧さよりも、早めの共有とフィードバックを優先する。


## Skill 利用の基本ルール

- 作業依頼を受けたら、最初に関連する Skill が存在するかを検討する。
- 適切な Skill がある場合は、Skill の利用を優先する。
- 自動的に Skill が呼び出されない場合でも、必要に応じて Skill 利用を提案する。
- Skill が適用できない場合は、その理由を簡潔に示した上で通常の対応を行う。


## 並列開発時のルール

- 並列開発（マルチエージェント開発）を行う場合、最初に `/project-orchestrator` を呼び出す。
- project-orchestrator が要件整理・技術選定・タスク分解を行い、適切な専門スキルに委譲する。


## 判断と確認

- 不明点・設計上の分岐・リスクがある場合は、選択肢を整理してユーザーに確認する。
- 一度決めた方針でも、前提が変わった場合は再確認を行う。
- 「確認 → 修正 → 前進」のループを前提として行動する。
